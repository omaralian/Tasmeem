Buttons

Every button case the framework supports. Variant (meaning) and style (appearance) are independent classes that compose: btn + a variant (primary, danger, …) + a style (outline, ghost). Build the stylesheet, then toggle the theme above to verify each case in light and dark.

Solid variants

The bare btn is the neutral default — no variant needed. Add a variant class to recolor.

Outline style

The outline style composes with any variant, so it borrows that variant's color. Hover to see it fill.

Ghost style

The ghost style is borderless and shows a faint variant tint on hover — the same variant classes apply.

States

Hover, active, and focus are interactive — hover a button, press and hold for active, or tab through them to see the variant-matched focus ring. Disabled buttons re-skin to neutral tokens and show a not-allowed cursor.

As different elements

The btn class renders consistently across <button>, <a>, and <input type="submit">.
